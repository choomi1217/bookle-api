# WordBook 설계 문서

이 디렉터리는 WordBook 앱의 설계 문서입니다. 상위 `CLAUDE.md`의 프로젝트 개요에서 이어지는 상세 설계와 기능별 버전 명세를 담습니다.

## 문서 구조

| 파일 | 내용 |
|---|---|
| [00-overview.md](./00-overview.md) | 전체 아키텍처, 기술 스택, 개발 로드맵 |
| [01-backend.md](./01-backend.md) | Spring Boot 서버 설계 |
| [02-mobile.md](./02-mobile.md) | React Native 앱 설계 |
| [03-ops.md](./03-ops.md) | 배포, 도메인, 운영 |
| [04-api.md](./04-api.md) | REST API 명세 |
| [book/book_v1.md](./book/book_v1.md) | 책 기능 v1 상세 명세 |
| [word/word_v1.md](./word/word_v1.md) | 단어 기능 v1 상세 명세 |

## 문서 버전 관리 규칙

- 기능 명세는 `book/`, `word/` 폴더에 `{기능}_v{번호}.md` 형태로 둔다.
- 기능을 개선할 때는 기존 파일을 직접 수정하지 않고 `_v2.md`, `_v3.md`를 **새로 추가**한다.
- 구버전 파일은 히스토리용으로 보존한다(삭제 금지).
- 상위 설계 문서(`00-*` ~ `04-*`)는 현재 운영 중인 내용을 반영하도록 갱신한다.

## 문서 작성 원칙

- "이게 왜 이렇게 됐는가"를 남긴다. 코드에서 읽히는 "무엇"은 최소화.
- 결정되지 않은 항목은 `> TODO:` 블록으로 명시하고, 결정되면 제거한다.
- 외부 링크는 영속성이 있는 공식 문서만 사용한다.
